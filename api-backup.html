ตัวอย่างโค้ด

Node.js (fetch)
const res = await fetch("https://api.thunder.in.th/v1/slip/verify", {
  method: "POST",
  headers: {
    Authorization: "Bearer <YOUR_API_KEY>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    bank: "kbank",
    transferred_at: "2025-08-21T10:15:33+07:00",
    amount: 590.00,
    ref_no: "KBANK-102233",
    slip_image: "<base64>",
  }),
});
const data = await res.json();
Python (requests)
import requests
r = requests.post(
  "https://api.thunder.in.th/v1/slip/verify",
  headers={"Authorization": "Bearer <YOUR_API_KEY>"},
  json={
    "bank": "scb",
    "transferred_at": "2025-08-21T10:15:33+07:00",
    "amount": 1290.00,
    "ref_no": "20250821101533",
    "slip_image": "<base64>",
  },
)
print(r.json())
PHP (cURL)
$ch = curl_init("https://api.thunder.in.th/v1/slip/verify");
curl_setopt_array($ch, [
  CURLOPT_POST => true,
  CURLOPT_HTTPHEADER => [
    "Authorization: Bearer <YOUR_API_KEY>",
    "Content-Type: application/json"
  ],
  CURLOPT_POSTFIELDS => json_encode([
    "bank" => "kbank",
    "transferred_at" => "2025-08-21T10:15:33+07:00",
    "amount" => 590.00,
    "ref_no" => "KBANK-102233",
    "slip_image" => "<base64>",
  ]),
  CURLOPT_RETURNTRANSFER => true
]);
$res = curl_exec($ch);

Webhooks

ตั้งค่า URL เพื่อรับเหตุการณ์ slip.verified, slip.duplicate, slip.failed และยืนยันลายเซ็นด้วย header Thunder-Signature

  • คืนสถานะ 2xx เสมอเมื่อตอบรับ
  • สามารถรีไทรตาม backoff ได้อัตโนมัติ

Rate Limits

ค่าเริ่มต้น: 5 req/sec ต่อ API key (ปรับเพิ่มได้ตามแพ็กเกจ)

โค้ดข้อผิดพลาด

400 Bad Request
ข้อมูลไม่ครบ/รูปแบบผิด
401 Unauthorized
API key ไม่ถูกต้อง/หมดอายุ
409 Conflict
ตรวจพบสลิปซ้ำ/รายการซ้ำ
429 Too Many Requests
เกิน rate limit
5xx Server Error
ข้อผิดพลาดจากระบบ ลองใหม่/รีไทร

ความปลอดภัย

  • ทุกคำขอผ่าน TLS
  • Webhook มีการลงลายเซ็น (HMAC) ตรวจสอบฝั่งผู้รับ
  • รองรับ IP allowlist และ Role-based Access (ตามแพ็กเกจ)
